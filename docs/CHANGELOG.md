# SIGNAL — 변경 기록

**기준:** 앱·문서 스냅샷 **1.0**. 이전 날짜별 세부 이력은 **2026-04-12**에 본 문서로 **한 번에 정리**했다. 이후 변경은 **날짜 섹션**만 아래에 추가한다.

## 2026-04-28

- **어드민 i18n:** 관리자 UI 언어 전환 시 `data-i18n`만 갱신되어 대시보드·목록 등 `innerHTML` 문구가 남던 문제를, 언어 변경 후 전역 데이터 리로드(`reloadAllAdminData`)로 즉시 다시 그리도록 수정했다.
- **앱 i18n:** 브리핑 주간 실적 칩·종목 상세 실적 행·어닝 필터 분기·공통 FY/Q 축약(`fiscalYearQuarterShort`)을 `locales`로 묶었고, 컨콜 요약 서비스(`services/concalls`)의 안내·에러 문구도 앱 언어를 따르도록 했다(캐시 키에 로케 포함).
- **어드민 설정 UX:** 버튼·탭·테이블 액션 버튼 크기와 폰트 위계를 맞추고, 뉴스 목록의 언어 필터는 제거해 편집 모달에서 언어를 전환하도록 정리했다.
- **Provider 설정:** Data Provider에서 모델 필드를 제거하고, LLM Provider 섹션에서 API Key·기본 모델·모델 프리셋을 함께 관리하도록 재구성했다.
- **뉴스 출처 관리:** 운영 정책은 출처 리스트 상단에 두고, 글로벌/코인 선택은 리스트 헤더 탭으로 옮겼으며, 출처 추가는 리스트 하단의 고정 입력 영역에서 `+`로 늘리고 저장 전 삭제할 수 있게 변경했다.
- **뉴스 편집:** 뉴스 목록은 원문 기준으로 단순화하고, 편집 모달에서 영어(원문)·한국어·일본어 탭으로 번역 결과를 확인·수정하도록 변경했다.
- **번역 설정:** 로케일별 설정은 Provider 선택만 담당하고, 실제 모델은 Provider 기본 모델을 따르도록 서버 Job 흐름을 정리했다.
- **뉴스 번역 표시:** mock 번역의 `[번역 대기]`/`[翻訳待ち]` 접두어 생성을 중단하고 기존 저장 데이터를 정리했으며, API 응답은 사용 가능한 번역이 없으면 원문을 반환하도록 보정했다.
- **어드민 다국어:** Provider·번역 설정·뉴스 편집·뉴스 출처의 주요 동적 문구를 i18n 키로 확장하고, 언어 변경 시 현재 화면을 다시 렌더링하도록 개선했다.
- **대시보드 UX:** 대시보드를 뉴스·유튜브·Job 최신 현황판으로 재구성하고, 표시 개수(기본 5개)를 브라우저 설정으로 저장하도록 추가했다.
- **오류 로그 UX:** 오류 메시지 컬럼을 제거하고 `오류 확인` 모달로 상세 정보와 실행 컨텍스트를 확인하도록 변경했다.
- **어드민 헤더:** 모바일 메뉴 버튼은 로고 쪽으로, 사이드바 접기 버튼은 메뉴 영역 상단으로 이동해 위치를 정리했다.
- **UI/UX 문서:** `docs/SIGNAL-ADMIN-UIUX.md`를 현행 어드민 기준 파일로 다시 정리해 레이아웃, 대시보드, 뉴스/출처, Provider/번역, 오류 로그 기준을 명문화했다.

## 2026-04-27

- **어드민 UI/UX:** 공식 어드민 가이드 기준으로 dark 운영 콘솔 톤, Header/Sidebar/PageHeader, 역할별 폰트 크기, 카드·테이블·폼·버튼 위계를 정리하고 데이터 초기화 화면을 카드형 선택·확인 패널로 개선. 대시보드·스케줄 관리·수집 현황 테이블의 Operation/도메인/Provider 컬럼을 분리해 의미를 통일하고, 시스템 하위 설정 화면을 카드형 설정 콘솔로 정리. 뉴스 출처를 마켓 리스트에서 별도 메뉴로 분리하고 검색 조건 영역의 그리드 정렬을 통일했으며, 뉴스 출처 화면 배치와 테이블 액션 버튼 크기를 조정.
- **어드민 UI/UX(문서 정렬):** `SIGNAL-ADMIN-UIUX.md`에 맞춰 주요 뷰에 `PageHeader`를 적용하고, 뉴스·캘린더·유튜브·Job 로그 검색 영역을 `filterBar` 레이아웃과 병용해 정리했다.
- **정적 자산:** `admin.css` / `app.js` 쿼리 버전과 빌드 pill을 갱신해 브라우저 캐시로 구버전이 남지 않게 했다.
- **Job 로그 선택:** 실행 로그 행 선택을 `jobKey`가 아닌 **`run.id` 기반 키**로 바꾸고, 일괄 재시도는 선택 행에서 **유니크 `jobKey`로 매핑**해 실행하도록 했다.
- **코드 구조:** 확인 모달과 토스트를 `admin/modal.js`, `admin/toast.js`로 분리해 `app.js`를 정리했다.
- **버그 수정:** 토스트 닫기(`data-toast-close`) 클릭이 동작하지 않던 문제, Job 설정 펼침 행 조회의 `CSS.escape` 의존, 대시보드에서 Job으로 이동 시 잘못된 스크롤 타깃(`.jobItem`)을 수정했다.

## 2026-04-12

### 제품·기능 (요약)

- **뉴스:** 글로벌·코인·한국 세그먼트, Finnhub 피드, 한국은 키워드 필터(`@/domain/news`). LLM 설정 시 **제목 번역**.
- **시세:** 관심·인기·시총·코인, Finnhub·Coingecko 연동, 탭 순서·목록 한도·글래스 등 로컬 설정.
- **컨콜·캘린더:** 실적·일정, `@/domain/concalls`·`@/domain/calendar`, API Ninjas·Finnhub 조합은 `services/concalls` 등 오케스트레이션.
- **유튜브:** 큐레이션·검색 부스트(`@/domain/youtube`), 기본 채널 시드는 `integrations/youtube/constants.ts`, 카드 링크는 `utils/openYoutube.ts`.
- **설정:** `app/settings.tsx` — 뉴스·유튜브·시세·캘린더·표시·알림.

### 코드·레이어 (요약)

- **`domain/`:** 뉴스·시세·컨콜·유튜브·테마·캘린더·더보기 허브 등 순수 규칙. **`@/domain/<영역>`** 배럴 import. 영역별 시드·순서는 **`domain/<영역>/constants.ts`**.
- **`integrations/<vendor>/`:** Finnhub·YouTube·Anthropic·OpenAI·AdMob·Coingecko·API Ninjas·expo-updates 등. 화면·서비스는 **`@/integrations/...`** 직접 import. 메모리 캐시는 통합 폴더 내 `*Cache.ts`. Finnhub는 `news`·`calendar`·`quotes`·`constants` 등으로 분리.
- **`services/`:** `env`, AsyncStorage 기반 설정, 오케스트레이션(레거시 re-export 최소화).
- **`hooks/`:** 공용 훅 + **`@/hooks`** 배럴. Context 전용 훅은 `contexts/`.
- **`constants/` (루트):** 테마 토큰(`SIGNAL`·`buildAppTheme`), 탭·세그먼트 UI, `newsSegment`, `megaCapUniverse` 등 **앱 전역** 정적 값.
- **문서:** `docs/SIGNAL-PRD.md`, `docs/AGENTS.md`, `docs/ARCHITECTURE.md`, 본 `CHANGELOG.md`. 레이어·상수·훅 규칙의 **단일 상세 기준**은 **`docs/ARCHITECTURE.md`**. 루트 `AGENTS.md`는 `docs/`로 연결하는 인덱스.
