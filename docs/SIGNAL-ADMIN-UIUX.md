## 1. 디자인 시스템 기초

### 1.1 컬러 팔레트

```
Primary (Signal Green)
  --color-primary-50:  #f0fdf4
  --color-primary-100: #dcfce7
  --color-primary-200: #bbf7d0
  --color-primary-500: #22c55e   ← 메인 브랜드 컬러
  --color-primary-600: #16a34a   ← 호버/액티브
  --color-primary-700: #15803d   ← 눌림 상태

Neutral (Gray Scale)
  --color-gray-50:  #f9fafb      ← 배경
  --color-gray-100: #f3f4f6      ← 사이드바 배경
  --color-gray-200: #e5e7eb      ← 경계선 (Border)
  --color-gray-400: #9ca3af      ← Placeholder / 비활성 텍스트
  --color-gray-600: #4b5563      ← 보조 텍스트
  --color-gray-800: #1f2937      ← 본문 텍스트
  --color-gray-900: #111827      ← 제목 텍스트

Semantic
  --color-success: #22c55e
  --color-warning: #f59e0b
  --color-error:   #ef4444
  --color-info:    #3b82f6

Surface
  --color-bg-page:    #f9fafb    ← 페이지 전체 배경
  --color-bg-card:    #ffffff    ← 카드/패널 배경
  --color-bg-sidebar: #1e2433    ← 사이드바 (다크)
```

### 1.2 타이포그래피

| 역할 | 크기 | 굵기 | 용도 |
|------|------|------|------|
| Page Title | 24px / 1.5rem | 700 | 페이지 대제목 |
| Section Title | 18px / 1.125rem | 600 | 섹션 헤더 |
| Card Title | 16px / 1rem | 600 | 카드/탭 제목 |
| Body | 14px / 0.875rem | 400 | 본문, 테이블 내용 |
| Label | 13px / 0.8125rem | 500 | 폼 라벨, 테이블 헤더 |
| Caption | 12px / 0.75rem | 400 | 보조 정보, 타임스탬프 |

- **기본 폰트**: `Pretendard`, `Inter`, `-apple-system` (순서대로 fallback)
- **라인 높이**: 본문 1.5, 제목 1.3
- **영문/숫자 데이터**: Tabular figures 설정 (`font-variant-numeric: tabular-nums`)

### 1.3 간격 시스템 (Spacing)

4px 기반 배수 사용 원칙

```
4px   → 아이콘-텍스트 사이, 뱃지 내부 패딩
8px   → 인풋 내부 패딩, 소형 요소 간격
12px  → 버튼 세로 패딩, 리스트 아이템 세로
16px  → 카드 내부 패딩, 섹션 간격
20px  → 카드 헤더 패딩
24px  → 페이지 콘텐츠 패딩
32px  → 주요 섹션 간격
40px  → 페이지 상단 여백
```

### 1.4 Border Radius

```
--radius-sm:  4px   → 뱃지, 태그
--radius-md:  8px   → 버튼, 인풋, 카드
--radius-lg:  12px  → 모달, 드롭다운 패널
--radius-xl:  16px  → 대형 카드
--radius-full: 9999px → 아바타, 토글, 원형 버튼
```

### 1.5 그림자 (Shadow)

```
--shadow-xs:  0 1px 2px rgba(0,0,0,0.05)              → 인풋, 버튼
--shadow-sm:  0 1px 3px rgba(0,0,0,0.10)              → 카드 기본
--shadow-md:  0 4px 6px rgba(0,0,0,0.07)              → 드롭다운
--shadow-lg:  0 10px 15px rgba(0,0,0,0.10)            → 모달
--shadow-xl:  0 20px 25px rgba(0,0,0,0.15)            → 알림 토스트
```

## 2. 레이아웃 구조

### 2.1 전체 레이아웃

```
┌────────────────────────────────────────────────┐
│           Global Header (60px)                 │
├──────────────┬─────────────────────────────────┤
│              │  Page Header (56px)             │
│   Sidebar    ├─────────────────────────────────┤
│   (240px)    │                                 │
│              │     Main Content Area           │
│              │     (padding: 24px)             │
│              │                                 │
└──────────────┴─────────────────────────────────┘
```

- **사이드바**: 고정 너비 240px, 좌측 고정(sticky)
- **콘텐츠 영역**: 나머지 전체 너비, 독립 스크롤
- **최소 지원 해상도**: 1280px × 800px
- **콘텐츠 최대 너비**: 1440px (중앙 정렬, 양 옆 padding 24px)

### 2.2 Page Header 영역 구조

```
┌─────────────────────────────────────────────────┐
│  ◀ [Breadcrumb]        [버튼] [버튼] [Primary]  │
│  Page Title                                     │
│  (보조 설명 텍스트 — 필요시)                      │
└─────────────────────────────────────────────────┘
```

- 높이: 콘텐츠에 따라 유동 (최소 56px)
- 하단 경계선: `1px solid var(--color-gray-200)`
- 우측 Action 버튼 그룹: 중요도 순 배치 (Primary 버튼은 맨 오른쪽)

## 3. 상단 헤더 (Global Header)

### 3.1 구조

```
┌───────────────────────────────────────────────────────────────────┐
│  [SIGNAL 로고]   [검색바]              [🔔] [도움말?] [프로필▼]  │
│  (24px 높이)     (최대 400px)                         (아바타)   │
└───────────────────────────────────────────────────────────────────┘
```

- **높이**: 60px 고정
- **배경색**: `#ffffff` (흰색) + 하단 `border-bottom: 1px solid var(--color-gray-200)`
- **z-index**: 50 (사이드바보다 높게)
- **좌측 로고**: 사이드바 너비(240px)와 동일 영역 내 수직 중앙 정렬
- **로고 클릭**: 대시보드(홈) 이동

### 3.2 글로벌 검색바

- 위치: 헤더 중앙 좌측 (로고 오른쪽, 전체 너비의 30% 내외)
- 플레이스홀더: `"메뉴, 기사, 사용자 검색..."` (⌘K 단축키 표시)
- 포커스 시: `shadow-md` 적용, border `--color-primary-500`
- 검색 결과: 드롭다운 패널 (최대 높이 400px, 스크롤)
  - 섹션 분리: `메뉴 바로가기 / 사용자 / 콘텐츠`
  - 각 항목: 아이콘 + 이름 + 보조설명
  - 결과 없음: "검색 결과가 없습니다" 안내

### 3.3 알림 아이콘 (🔔)

- 아이콘 우측 상단: 읽지 않은 알림 수 뱃지 (최대 표시: 99+)
- 뱃지 색상: `--color-error` (빨강)
- 클릭 시: 알림 드롭다운 패널 (너비 360px)
  - 상단: "알림" 제목 + "모두 읽음" 버튼
  - 알림 리스트: 아이콘 + 제목 + 시간 + 읽음 여부
  - 하단: "전체 알림 보기" 링크

### 3.4 프로필 드롭다운

- 표시: 아바타 이미지 (없으면 이니셜 원형) + 이름 + 역할
- 클릭 시 드롭다운 메뉴:
  ```
  ─────────────────
  👤  내 프로필
  ⚙️  계정 설정
  ─────────────────
  🌐  언어 설정
  ─────────────────
  🚪  로그아웃
  ─────────────────
  ```
- 드롭다운 너비: 200px
- 위치: 헤더 우측 끝에서 8px 안쪽 정렬, 헤더 하단 8px 아래에서 시작

## 4. 사이드 메뉴 (Sidebar Navigation)

### 4.1 구조 및 스타일

- **너비**: 240px (고정), 접힘(Collapsed) 시 64px
- **배경**: `#1e2433` (다크 네이비)
- **텍스트**: `#cbd5e1` (기본), `#ffffff` (활성)
- **액티브 아이템 배경**: `rgba(34, 197, 94, 0.15)` + 좌측 `3px solid #22c55e`
- **호버 배경**: `rgba(255,255,255,0.06)`

### 4.2 메뉴 항목 구조

```
┌──────────────────────────────────┐
│  ● SIGNAL Admin       [< 접기]   │  ← 상단 브랜드 영역 (60px)
├──────────────────────────────────┤
│  [검색] 메뉴 검색...             │  ← 사이드바 내 검색 (선택)
├──────────────────────────────────┤
│  대시보드 그룹                   │  ← 섹션 라벨 (10px 캡션, 대문자)
│  🏠  대시보드                    │
├──────────────────────────────────┤
│  콘텐츠 관리                     │
│  📰  뉴스 관리                   │
│  📌  주요 종목 관리              │
│  🏷️  카테고리 관리               │
├──────────────────────────────────┤
│  사용자 관리                     │
│  👥  사용자                      │
│  🔑  권한/역할                   │
├──────────────────────────────────┤
│  시스템                          │
│  📊  통계/분석                   │
│  ⚙️  시스템 설정                 │
│  📋  로그                        │
└──────────────────────────────────┘
│  [관리자 이름]  [로그아웃 아이콘] │  ← 하단 고정 (64px)
```

### 4.3 메뉴 항목 규격

- **높이**: 44px
- **패딩**: 좌 16px, 우 12px
- **아이콘**: 20×20px, 텍스트 왼쪽 12px 간격
- **서브메뉴**: 들여쓰기 36px, 높이 40px, 아이콘 없음(점 또는 대시)
- **섹션 라벨**: 10px uppercase 텍스트, `--color-gray-400`, 상단 패딩 20px

### 4.4 서브메뉴 펼침/접힘

- 서브메뉴 있는 항목: 우측에 `▾` 화살표 아이콘
- 클릭 시: 부드러운 height 애니메이션 (200ms ease)
- 펼침 상태: 화살표 180° 회전
- 현재 페이지의 상위 메뉴는 항상 펼침 상태 유지

### 4.5 뱃지 (Badge)

- 알림/카운트가 필요한 메뉴 오른쪽에 표시
- 예: "뉴스 관리" 오른쪽에 "신규 12" 뱃지
- 스타일: `--color-primary-500` 배경, 흰색 텍스트, `--radius-full`

### 4.6 접힘(Collapsed) 모드

- 너비: 64px, 아이콘만 표시
- 호버 시: 툴팁으로 메뉴명 표시 (우측에 팝업)
- 토글 버튼: 헤더 좌측 또는 사이드바 우측 상단

## 5. 메뉴별 화면 구성

> 각 메뉴 화면은 **Page Header + 콘텐츠 영역** 구조를 기본으로 합니다.

### 5.1 대시보드 (Dashboard)

**목적**: 핵심 지표를 한눈에 파악

**레이아웃 구조**:
```
[Page Header: 대시보드]                    [날짜 필터 ▼] [새로고침]

┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐
│ KPI 카드 │ │ KPI 카드 │ │ KPI 카드 │ │ KPI 카드 │   ← 상단 KPI (4열)
└─────────┘ └─────────┘ └─────────┘ └─────────┘

┌──────────────────────────┐ ┌──────────────────┐
│   메인 차트 (2/3)         │ │  보조 차트 (1/3)  │   ← 차트 영역
└──────────────────────────┘ └──────────────────┘

┌──────────────────────────┐ ┌──────────────────┐
│  최근 뉴스 리스트         │ │  빠른 링크/액션   │   ← 하단 위젯
└──────────────────────────┘ └──────────────────┘
```

**KPI 카드 규격**:
- 흰색 배경, `--shadow-sm`, `--radius-lg`
- 상단: 제목 (label) + 아이콘
- 중앙: 메인 수치 (24px Bold)
- 하단: 전일/전주 대비 증감 (▲▼ + 퍼센트, 컬러 구분)
- 클릭 시 해당 상세 메뉴로 이동

### 5.2 뉴스 관리

**탭 구조**:
```
[전체 뉴스] [수집 설정] [카테고리 매핑] [키워드 필터]
```

#### 탭 1 — 전체 뉴스

```
[Page Header: 뉴스 관리]                  [수동 수집 실행] [+ 뉴스 등록]

[검색: 제목/내용...] [소스▼] [카테고리▼] [상태▼] [날짜범위] [초기화]

┌──────────────────────────────────────────────────────────────────┐
│ □ | 제목         | 소스    | 카테고리 | 상태   | 발행일    | 액션 │
├──────────────────────────────────────────────────────────────────┤
│ □ | 뉴스 제목... | Reuters | 시장     | 발행   | 04-27     | ⋮   │
│   |  (보조: 요약 1줄)                                            │
└──────────────────────────────────────────────────────────────────┘

[선택된 N개: [발행] [숨김] [삭제]]          [1 2 3 ... 10] [50개씩▼]
```

- 액션 버튼(⋮): 드롭다운 → `보기 / 수정 / 발행 / 숨김 / 삭제`
- 상태 뱃지: 발행(green) / 대기(yellow) / 숨김(gray) / 오류(red)
- 행 클릭: 상세 슬라이드오버(Side Panel) 또는 상세 페이지 이동

#### 탭 2 — 수집 설정

```
[+ 소스 추가]

┌──────────────────────────────────────────────────────────┐
│ 소스명       | URL         | 유형    | 주기   | 상태 | 액션 │
├──────────────────────────────────────────────────────────┤
│ Reuters RSS  | rss.reuters | RSS     | 15분   | 활성 | ⋮   │
│ Finnhub API  | api.finn... | API     | 5분    | 활성 | ⋮   │
└──────────────────────────────────────────────────────────┘
```

#### 탭 3 — 카테고리 매핑

- 드래그앤드롭 또는 셀렉트박스로 키워드 ↔ 카테고리 매핑 설정

#### 탭 4 — 키워드 필터

- 차단 키워드 목록 관리 (추가/삭제)
- 태그 형식으로 표시

**탭 구조**:
```

```

#### 탭 1 — 사용자 목록

```

└───────────────────────────────────────────────────────────────────┘
```

- 행 클릭: 사용자 상세 Side Panel 열림

## 6. 탭(Tab) 구조 기준

### 6.1 탭 사용 기준

| 상황 | 처리 방법 |
|------|----------|
| 동일 데이터의 다른 뷰 | 탭 사용 |
| 독립적인 기능 그룹 | 탭 사용 |
| 단순 필터링 | 필터 컨트롤 사용 (탭 X) |
| 3개 이하 항목 | 탭 또는 토글 버튼 |
| 7개 이상 항목 | 드롭다운 탭 또는 메뉴 분리 검토 |

### 6.2 탭 스타일 규격

**Line Tab** (페이지 내 주요 탭 — 권장):
```
[탭1]  [탭2]  [탭3]
──────                ← 활성 탭 하단: 2px solid --color-primary-500
────────────────────  ← 전체 하단 구분선: 1px solid --color-gray-200
```
- 높이: 44px
- 패딩: 좌우 16px
- 폰트: 14px, 활성 600 / 비활성 400
- 색상: 활성 `--color-primary-600` / 비활성 `--color-gray-500`

**Pill Tab** (카드 내부 소형 탭):
```
[탭1] [탭2▐활성]  ← 활성: 배경 --color-gray-100, 폰트 600
```
- 높이: 32px
- 패딩: 좌우 12px
- `--radius-md` 적용

### 6.3 탭 내 콘텐츠 전환

- 전환 애니메이션: `opacity 0 → 1`, 150ms ease
- URL 파라미터 반영: `?tab=수집설정` (브라우저 뒤로가기 대응)
- 탭 전환 시 스크롤 위치 유지 (탭 하단부터)

### 6.4 탭 내 빈 상태

- 각 탭은 독립적인 빈 상태 처리
- 탭 제목 옆 카운트 뱃지: `[탭명 (0)]` 로 데이터 없음 암시

## 7. 테이블 & 리스트 컴포넌트

### 7.1 테이블 기본 규격

```
헤더행
  - 높이: 44px
  - 배경: --color-gray-50
  - 폰트: 13px, 500 (Medium)
  - 색상: --color-gray-600
  - 정렬: 기본 좌측, 숫자 우측, 액션 중앙
  - 정렬 가능 컬럼: ▲▼ 아이콘 표시 (클릭 시 토글)

데이터행
  - 높이: 52px (다중행 데이터: 최소 52px, 자동 확장)
  - 배경: 기본 white, 호버 --color-gray-50
  - 폰트: 14px, 400
  - 하단 구분선: 1px solid --color-gray-100
  - 선택 시 배경: --color-primary-50
```

### 7.2 체크박스 선택

- 헤더 체크박스: 전체 선택/해제
- 부분 선택 시 헤더: 인디터미네이트(-) 상태
- 선택 시 상단에 일괄 액션 바 표시:
  ```
  ✓ 5개 선택됨  [발행] [숨김] [삭제]  [선택 해제]
  ```

### 7.3 페이지네이션

```
[처음] [이전]  1  2  3  ...  10  [다음] [마지막]    페이지당 [50개 ▼]
총 487건 중 1-50번째
```
- 표시 페이지 번호: 현재 기준 앞뒤 2개 + 처음/마지막
- 페이지당 표시 건수: 25 / 50 / 100 선택
- 총 건수 항상 표시

### 7.4 컬럼 설정

- 우측 상단 `[열 설정 ⚙]` 버튼: 표시할 컬럼 체크박스 선택
- 설정은 로컬스토리지에 사용자별 저장

### 7.5 행 확장 (Row Expansion)

- 요약 정보는 테이블, 상세 내용은 행 하단 확장 패널로 표시
- 확장 버튼: 행 좌측 `▶` 아이콘

## 8. 폼(Form) 컴포넌트

### 8.1 레이아웃 원칙

- 기본: 1열 레이아웃 (최대 너비 720px)
- 관련 필드 묶음: `fieldset` 또는 시각적 섹션 구분 (회색 배경 or 선)
- 라벨 위치: 인풋 상단 (Top-label 방식)
- 필수 항목: 라벨 우측 `*` (빨간색)

### 8.2 인풋 규격

```
상태        | 테두리              | 배경      | 텍스트
──────────────────────────────────────────────────────
기본        | --color-gray-300   | white     | --color-gray-800
포커스      | --color-primary-500 | white    | --color-gray-800
비활성      | --color-gray-200   | --gray-50 | --color-gray-400
오류        | --color-error       | #fff5f5  | --color-gray-800
성공        | --color-success    | white     | --color-gray-800
```

- 높이: 40px (기본), 36px (소형), 48px (대형)
- 패딩: 좌우 12px
- `--radius-md` 적용
- 오류 메시지: 인풋 하단 4px 아래, 12px 빨간 텍스트

### 8.3 폼 버튼 배치

```
[취소]  [임시저장]  [저장/확인]   ← 폼 하단 우측 정렬
```
- Primary 버튼: 맨 오른쪽
- 취소: 텍스트 버튼 또는 Ghost 버튼
- 간격: 버튼 간 8px

### 8.4 버튼 스타일 정의

| 유형 | 배경 | 텍스트 | 테두리 | 용도 |
|------|------|--------|--------|------|
| Primary | --color-primary-500 | white | 없음 | 주요 액션 |
| Secondary | white | --color-primary-600 | primary | 보조 액션 |
| Danger | --color-error | white | 없음 | 삭제/위험 |
| Ghost | transparent | --color-gray-600 | --gray-300 | 취소 |
| Text | transparent | --color-primary-600 | 없음 | 링크성 액션 |

- 높이: 36px (소) / 40px (기본) / 44px (대)
- 패딩: 좌우 16px
- `--radius-md` 적용
- 로딩 상태: 버튼 내 스피너 표시, 텍스트 숨김, 비활성화

### 8.5 Select / Dropdown

- 기본 인풋과 동일 스타일
- 드롭다운 패널: `--shadow-md`, `--radius-lg`, 최대 높이 240px + 스크롤
- 검색 가능 Select: 패널 상단에 검색 인풋 포함
- 다중 선택: 태그 형태로 인풋 내부에 표시

## 9. 팝업 & 모달 기준

### 9.1 모달 크기 분류

| 크기 | 너비 | 용도 |
|------|------|------|
| Small | 400px | 단순 확인/경고 (Confirm) |
| Medium | 560px | 소형 폼, 상세 정보 보기 |
| Large | 720px | 복잡한 폼, 멀티 필드 입력 |
| XLarge | 960px | 미리보기, 복잡한 설정 |
| Full | 100vw - 48px | 대형 데이터 편집, 에디터 |

### 9.2 모달 구조

```
┌──────────────────────────────────────────────────┐
│  모달 제목                               [✕]     │  ← Header (56px)
│  (보조 설명 텍스트, 선택)                         │    border-bottom
├──────────────────────────────────────────────────┤
│                                                  │
│   콘텐츠 영역                                    │  ← Body (스크롤)
│   (padding: 24px)                               │    max-height: 60vh
│                                                  │
├──────────────────────────────────────────────────┤
│                       [취소]  [확인/저장]        │  ← Footer (64px)
└──────────────────────────────────────────────────┘
```

**필수 규칙**:
- 오버레이: `rgba(0, 0, 0, 0.5)`, z-index 100
- 모달: z-index 101
- 등장 애니메이션: `scale(0.95) → scale(1)` + `opacity 0→1`, 200ms ease-out
- ESC 키: 모달 닫기
- 오버레이 클릭: 기본 닫기 (데이터 입력 모달은 확인 후 닫기)
- 중첩 모달: 최대 2단계까지 허용 (z-index 102)

### 9.3 Confirm 모달 (Small — 400px)

```
┌──────────────────────────────┐
│  ⚠️ 삭제 확인          [✕]  │
├──────────────────────────────┤
│                              │
│  정말로 삭제하시겠습니까?    │
│  이 작업은 되돌릴 수 없습니다. │
│                              │
├──────────────────────────────┤
│             [취소]  [삭제]   │
└──────────────────────────────┘
```

- 삭제 확인: `[삭제]` 버튼 = Danger 스타일 (빨강)
- 단순 확인: `[확인]` 버튼 = Primary 스타일
- 아이콘: ⚠️ 경고 / ℹ️ 안내 / ✅ 성공 / 🗑️ 삭제

### 9.4 폼 모달 (Medium~Large)

- 헤더에 현재 작업 명시: `"뉴스 등록"`, `"사용자 수정"` 등
- 필수 필드 상단 배치
- 폼 유효성 검사: 저장 클릭 시 일괄 검사, 오류 필드 포커스 이동
- 저장 완료: 모달 닫힘 + 성공 토스트 알림

### 9.5 상세 보기 모달 (View Modal)

- 편집 불가한 정보 표시 전용
- 우측 상단: `[수정하기]` 버튼 → 폼 모달로 전환
- 레이블: 12px gray / 값: 14px dark

### 9.6 Side Panel (Slide Over)

테이블에서 상세 정보를 보면서 목록을 유지해야 할 때 사용

```
┌──────────────────────────────┐
│  상세 정보             [✕]   │
├──────────────────────────────┤
│                              │
│   콘텐츠                     │
│   (독립 스크롤)              │
│                              │
├──────────────────────────────┤
│  [수정] [삭제]               │
└──────────────────────────────┘
```

- 너비: 480px (기본), 640px (대형)
- 우측에서 슬라이드: `translateX(100%) → translateX(0)`, 250ms ease
- 오버레이: `rgba(0,0,0,0.3)` (모달보다 연하게)
- 오버레이 클릭: 패널 닫기

### 9.7 모달 사용 결정 트리

```
액션이 필요한가?
    ├── YES: 새 페이지 이동이 필요한 복잡한 작업인가?
    │         ├── YES → 별도 페이지로 이동
    │         └── NO  → 폼 모달 사용
    └── NO: 정보만 보는가?
              ├── 목록과 함께 봐야 하는가?
              │     ├── YES → Side Panel
              │     └── NO  → 상세 보기 모달
              └── 단순 확인/경고 → Confirm 모달
```

## 10. 알림 & 피드백 컴포넌트

### 10.1 Toast 알림

- 위치: 화면 우측 하단 (bottom: 24px, right: 24px)
- 최대 동시 표시: 3개 (스택)
- 자동 닫힘: 기본 4초 (에러: 수동 닫기)
- 너비: 360px

```
┌──────────────────────────────────────┐
│  ✅  저장되었습니다.           [✕]  │  ← Success (초록 좌측 바)
└──────────────────────────────────────┘

┌──────────────────────────────────────┐
│  ❌  오류가 발생했습니다.      [✕]  │  ← Error (빨간 좌측 바)
│      서버에 연결할 수 없습니다.      │
└──────────────────────────────────────┘
```

| 유형 | 좌측 바 색상 | 아이콘 |
|------|------------|--------|
| Success | --color-success | ✅ |
| Error | --color-error | ❌ |
| Warning | --color-warning | ⚠️ |
| Info | --color-info | ℹ️ |

### 10.2 인라인 알림 (Alert Banner)

- 페이지 또는 카드 내 상단에 표시
- 닫기 가능 여부 옵션 (중요 공지는 닫기 X)

```
┌────────────────────────────────────────────────────────┐
│  ⚠️  일부 뉴스 수집 소스가 오류 상태입니다. [설정 보기]   [✕]  │
└────────────────────────────────────────────────────────┘
```

### 10.3 로딩 상태

- **전체 페이지 로딩**: 중앙 스피너 + 배경 오버레이 (초기 진입 시)
- **테이블 로딩**: 스켈레톤 행 (5행) 표시
- **버튼 로딩**: 버튼 내 소형 스피너 + 비활성화
- **카드 로딩**: 스켈레톤 카드
- 스켈레톤 색상: `--color-gray-200`, 애니메이션: shimmer 효과

## 11. 빈 상태 & 에러 상태

### 11.1 빈 상태 (Empty State)

```
        [일러스트 또는 아이콘]
           (64×64px)

        데이터가 없습니다
        (16px, --color-gray-500)

        설명 문구 (선택)
        (14px, --color-gray-400)

        [+ 첫 항목 추가하기]  ← CTA 버튼 (선택)
```

- 검색 결과 없음: "검색 결과가 없습니다. 다른 키워드로 시도해보세요."
- 필터 결과 없음: "조건에 맞는 항목이 없습니다. [필터 초기화]"
- 오류로 인한 빈 상태: 에러 상태와 구분

### 11.2 에러 상태

```
        ⚠️ (48px 아이콘)

        데이터를 불러올 수 없습니다

        잠시 후 다시 시도해주세요.

        [다시 시도]
```

- 페이지 에러 (500): 전체 에러 페이지
- 컴포넌트 에러: 해당 컴포넌트 영역 내에만 표시

## 13. 접근성 기준

- **키보드 내비게이션**: 모든 인터랙티브 요소 Tab 접근 가능
- **포커스 스타일**: `outline: 2px solid --color-primary-500`, offset 2px
- **색상 대비**: WCAG AA 기준 이상 (4.5:1 이상)
- **ARIA 레이블**: 아이콘만 있는 버튼에 `aria-label` 필수
- **모달**: 열릴 때 첫 요소로 포커스 이동, `role="dialog"`, `aria-modal="true"`
- **상태 변화**: `aria-live="polite"` 로 스크린리더에 알림
- **폼 오류**: `aria-describedby`로 오류 메시지 연결
