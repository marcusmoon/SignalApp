# SIGNAL Admin UI/UX 기준

> 최종 수정: 2026-04-28
> 이 문서는 SIGNAL 어드민의 현행 UI/UX 기준 파일입니다. 신규 화면과 기존 화면 개선은 이 문서를 우선 기준으로 삼습니다.

## 핵심 원칙

- 어드민은 현황 파악과 운영 변경이 빠른 업무 도구다. 장식보다 정보 밀도, 스캔 속도, 일관된 액션 배치가 우선이다.
- 첫 화면인 대시보드는 실행 도구가 아니라 현황판이다. 뉴스, 유튜브, Job 같은 주요 영역의 최신 항목과 이상 상태를 보여주고, 상세 작업은 각 영역으로 이동해 처리한다.
- 카드 안에 카드를 중첩하지 않는다. 반복 아이템, 모달, 독립 도구만 카드로 감싸고 섹션은 단순한 레이아웃으로 둔다.
- 버튼, 입력, 탭, 뱃지는 같은 역할이면 같은 크기와 폰트 무게를 유지한다. 작은 테이블 액션은 `compactBtn`, 아이콘 액션은 `iconBtn`을 쓴다.
- 목록 화면은 원본 데이터를 중심으로 보여주고, 번역/오류/긴 본문처럼 공간을 많이 쓰는 정보는 모달 또는 상세 화면에서 편집한다.

## 레이아웃

- 상단 헤더에는 브랜드, 메뉴 토글, 글로벌 검색, 언어/시간/프로필 도구를 둔다.
- 사이드바 접기 버튼은 메뉴 영역 가까이에 둔다. 사용자가 메뉴 폭을 조절한다고 인지하기 쉬워야 한다.
- 사이드바는 메뉴 그룹을 짧게 유지하고, 한국어 UI에서는 `Provider`를 `프로바이더`처럼 언어별 표기로 노출한다.
- 본문은 `24px` 내외의 여백과 단일 스크롤 흐름을 기본으로 한다. 테이블이 넓으면 가로 스크롤을 허용하되, 필터와 저장 액션은 화면에서 찾기 쉬워야 한다.

## 대시보드

- 대시보드는 상단 통계와 영역별 최신 목록으로 구성한다.
- 최신 뉴스, 최신 유튜브, Job 현황은 각각 초기 `5개`를 보여준다. 표시 개수는 대시보드 안에서 `3 / 5 / 10` 중 선택할 수 있고 브라우저에 저장한다.
- Job 영역에는 실행 버튼을 두지 않는다. 설정 또는 로그 상세로 이동하는 액션만 제공한다.
- 실패/주기 초과 Job 수는 Job 현황 헤더에서 즉시 확인할 수 있어야 한다.

## 뉴스

- 뉴스 목록은 영어 원문 기준으로 표시한다.
- 번역 편집은 `편집` 모달에서 처리한다. 모달 안에는 `English(Original) / 한국어 / 日本語` 탭을 두고 각 탭에서 제목과 요약을 편집한다.
- 번역 결과가 없거나 mock 대기 문구에 해당하는 값이면 저장/노출하지 않는다. 앱과 API는 동일하게 번역이 없을 때 원문을 fallback으로 사용한다.
- 상단 검색 필터에는 언어 선택을 두지 않는다. 언어별 확인과 편집은 모달에서 한다.

## 뉴스 출처

- 운영 정책은 출처 리스트 상단에 둔다. 글로벌/코인 신규 출처 자동 노출과 숨김 포함 보기를 여기서 조정한다.
- 카테고리 전환은 출처 리스트 헤더의 탭으로 둔다.
- 출처 추가는 리스트 하단에 고정된 추가 영역에서 한다. `+` 버튼으로 입력 행을 늘리고, 저장 전 임시 행은 삭제할 수 있다.
- 기존 출처는 삭제보다 숨김/복구를 우선한다. 별칭은 출처별 모달에서 관리한다.

## 설정

- `Data Provider`에는 API Key와 활성 상태만 둔다. 수집 Provider에 모델 정보는 두지 않는다.
- LLM 모델 프리셋과 기본 모델은 `Provider > LLM Provider`에서 관리한다.
- 번역 설정은 로케일별 사용 여부, 자동 번역 여부, Provider 선택을 관리한다. 실제 모델은 선택된 Provider의 기본 모델을 따른다.
- 향후 뉴스 외 유튜브/기타 콘텐츠 번역이 늘어날 수 있으므로 번역 설정의 문구와 구조는 특정 도메인에 과하게 묶지 않는다.

## 오류와 로그

- 오류 로그 테이블은 요약 컬럼만 보여준다. 긴 오류 메시지는 `오류 확인` 모달에서 본다.
- 오류 모달은 메시지, 스택/원문, 실행 컨텍스트를 분리한다.
- 실행 로그 상세에서는 재시도나 삭제처럼 실제 운영 액션을 제공하고, 대시보드에서는 이동만 제공한다.

## 다국어

- 새 UI 문자열은 `ko / en / ja` 키를 함께 추가한다.
- 화면 전환 없이 언어를 바꾸면 현재 화면의 동적 영역도 다시 렌더링되어야 한다.
- 영문 고유명사라도 메뉴나 설명성 라벨은 가능한 한 현재 언어에 맞춰 표시한다.

## 컴포넌트 기준

- 탭은 같은 화면의 범주 전환에만 쓴다. 상태 필터에는 컴팩트 탭을 사용할 수 있다.
- 테이블 액션은 우측 끝에 모으고, 동일 테이블 안에서는 액션 순서를 유지한다.
- 모달은 제목, 보조 메타, 본문, 하단 액션 순서로 구성한다. 닫기, 취소, Escape, backdrop 클릭을 지원한다.
- 버튼 텍스트는 짧게 쓴다. 익숙한 조작은 아이콘 버튼을 쓰되 `title`을 제공한다.
- 모바일/좁은 화면에서는 필터와 그리드를 한 열로 내려 정보가 겹치지 않게 한다.
