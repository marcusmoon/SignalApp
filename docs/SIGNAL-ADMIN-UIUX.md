# SIGNAL Admin UI/UX 기준

> 최종 수정: 2026-05-01
> 이 문서는 SIGNAL 어드민의 현행 UI/UX 기준 파일입니다. 신규 화면과 기존 화면 개선은 이 문서를 우선 기준으로 삼습니다.

## 핵심 원칙

- 어드민은 현황 파악과 운영 변경이 빠른 업무 도구다. 장식보다 정보 밀도, 스캔 속도, 일관된 액션 배치가 우선이다.
- 첫 화면인 대시보드는 실행 도구가 아니라 현황판이다. 뉴스, 유튜브, Job 같은 주요 영역의 최신 항목과 이상 상태를 보여주고, 상세 작업은 각 영역으로 이동해 처리한다.
- 카드 안에 카드를 중첩하지 않는다. 반복 아이템, 모달, 독립 도구만 카드로 감싸고 섹션은 단순한 레이아웃으로 둔다.
- 버튼, 입력, 탭, 뱃지는 같은 역할이면 같은 크기와 폰트 무게를 유지한다. 작은 테이블 액션은 `compactBtn`, 아이콘 액션은 `iconBtn`을 쓴다.
- 목록 화면은 원본 데이터를 중심으로 보여주고, 번역/오류/긴 본문처럼 공간을 많이 쓰는 정보는 모달 또는 상세 화면에서 편집한다.

## 레이아웃

- 상단 헤더에는 브랜드, 메뉴 토글, 글로벌 검색, 언어/시간/프로필 도구를 둔다. **데스크톱(1024px 이상)** 기준 배치이며, 좁은 화면에서는 아래 「모바일」을 따른다.
- 사이드바 접기/펼치기는 사이드바 경계에 **투명 거터(gutter)**를 두고, hover 시 버튼을 노출한다. 평소에는 숨기고(시각적 잡음 최소화), 클릭은 안정적으로 동작해야 한다. **gutter·접힘 아이콘 모드는 데스크톱**에서만 쓴다.
- 사이드바는 메뉴 그룹을 짧게 유지하고, 한국어 UI에서는 `Provider`를 `프로바이더`처럼 언어별 표기로 노출한다.
- 본문은 `24px` 내외의 여백과 단일 스크롤 흐름을 기본으로 한다. 테이블이 넓으면 가로 스크롤을 허용하되, 필터와 저장 액션은 화면에서 찾기 쉬워야 한다.

## 모바일

- `1024px` 미만: 본문 그리드는 **1열**. 사이드바는 **햄버거 드로어**(`position: fixed`, 딤 탭으로 닫힘). 드로어 안 네비는 **세로 목록**이며, 가로 칩 한 줄로 메뉴를 쭉 붙이지 않는다. production `admin.css`의 2열·sticky 사이드·`grid-template-columns: 240px 1fr` 규칙은 **`@media (min-width: 1024px)`** 안에만 둬서 모바일 규칙을 덮어쓰지 않게 한다.
- 헤더는 **세로 스택**: 1행에 햄버거+브랜드(로고 옆 서브타이틀은 숨겨 공간 확보), 2행 `topTools`에서 **글로벌 검색 입력을 전체 폭**으로 올리고, 그 아래 줄에 알림·도움말·프로필·언어·타임존·세션·로그아웃이 줄바꿈된다. 헤더 실제 높이는 JS `ResizeObserver`로 재서 `--admin-header-h`에 넣고, 드로어의 `top`/`height`와 레이아웃 `min-height`가 이 값을 따른다.
- 페이지 헤더는 제목/설명과 액션을 세로로 쌓는다. 탭·새로고침처럼 가로로 긴 액션은 내부 가로 스크롤을 허용한다.
- PC와 모바일은 별도 코드베이스가 아니라 같은 정보 구조에서 **표현과 조작 밀도**만 다르게 둔다. PC는 비교와 일괄 작업, 모바일은 확인과 좁은 수정, 즉시 액션을 우선한다.
- 필터 그룹은 모바일에서 기본 접힘 패널로 둔다. 조건을 펼치면 한 열로 표시하고, `480px` 이하에서는 날짜 범위, 검색어, 버튼을 모두 전체 폭으로 내려 겹침을 막는다.
- 운영용 원장 테이블은 카드 폭 안에서 가로 스크롤한다. 뉴스처럼 사용자가 자주 훑는 목록은 모바일에서 라벨이 붙은 카드형 행으로 바꿔 가로 스크롤 의존도를 낮춘다.
- 대시보드 카드와 콘텐츠 리스트는 한 열로 내려가며, 액션 버튼은 터치하기 쉬운 폭을 유지한다.
- 설정·Provider·초기화처럼 설명이 긴 화면은 모바일에서 설명문을 2줄 이내로 줄이고, 실제 입력/저장 컨트롤을 먼저 찾을 수 있게 한다.
- 수집 Job과 실행 이력은 모바일에서 표를 그대로 축소하지 않는다. Job 목록과 실행 이력은 카드형으로 보여주고, 실패·실행 중·멈춤 의심 상태는 왼쪽 상태 라인과 pill로 표시한다.
- 콘텐츠 수집 관리 흐름은 `수집 Job` → `실행 모니터링` → `인사이트 결과` → `실패/오류 로그` 순서로 둔다. 상단 흐름 메뉴는 카드 전체가 클릭 가능해야 하며, 활성 카드의 텍스트 대비가 테마와 무관하게 충분해야 한다.
- 모바일 화면은 세로 스크롤이 길어질 수 있으므로 일정 거리 이상 내려가면 하단 우측에 작은 `맨 위로` 플로팅 버튼을 제공한다. 버튼은 safe-area를 피하고 주요 액션을 가리지 않아야 한다.

## 대시보드

- 대시보드는 데이터 상태, 콘텐츠 현황, Job 현황으로 구성한다.
- 최신 뉴스와 최신 유튜브는 **콘텐츠 현황**으로 묶고, 초기 `5개`를 보여준다. 표시 개수는 섹션 안에서 `3 / 5 / 10` 중 선택할 수 있고 브라우저에 저장한다.
- 앱 첫 화면은 원시 피드보다 **오늘의 시그널**을 우선 노출한다. 서버 인사이트가 있으면 뉴스·유튜브·시세를 연결한 짧은 판단 카드로 보여주고, 관련 원문으로 넘어갈 수 있게 한다.
- 어드민에는 생성된 인사이트를 검수하는 **오늘의 시그널** 화면을 둔다. 운영자는 생성 시각, 점수, push 후보 여부, 연결 원문, LLM 준비 상태를 확인할 수 있어야 한다.
- Job 영역에는 실행 버튼을 두지 않는다. 설정 또는 로그 상세로 이동하는 액션만 제공한다.
- 실패/주기 초과/실행 중/멈춤 의심 Job 수는 Job 현황 헤더에서 즉시 확인할 수 있어야 한다.
- 데이터 상태처럼 여러 상태를 같은 화면에서 비교하는 카드는 기본 외곽 테두리와 섹션 프레임을 accent 계열로 통일한다. 실패/경고/실행 중 같은 의미 색은 카드 전체 테두리를 바꾸기보다 왼쪽 3~4px 세로 라인과 작은 상태 pill로 표시한다.
- 상태 옆의 보조 액션(예: 로그)은 일반 버튼보다 작은 pill 버튼으로 붙여 시각적 무게를 줄인다.

## 뉴스

- 뉴스 목록은 영어 원문 기준으로 표시한다.
- 번역 편집은 `편집` 모달에서 처리한다. 모달 안에는 `English(Original) / 한국어 / 日本語` 탭을 두고 각 탭에서 제목과 요약을 편집한다.
- 해시태그는 뉴스 편집 모달에서 관리한다. 자동 태그는 AI 번역 결과를 따르고, 수동 저장 시 자동 번역이 덮어쓰지 않게 고정한다.
- 번역 결과가 없거나 mock 대기 문구에 해당하는 값이면 저장/노출하지 않는다. 앱과 API는 동일하게 번역이 없을 때 원문을 fallback으로 사용한다.
- 상단 검색 필터에는 언어 선택을 두지 않는다. 언어별 확인과 편집은 모달에서 한다.
- 모바일 뉴스 목록은 표 형태보다 카드형 행을 우선한다. 시간, 출처, 카테고리, 번역 상태는 작은 라벨로 보조화하고 제목과 원문 이동/편집 액션을 더 크게 둔다.

## 유튜브

- 유튜브 관리는 썸네일이 주 정보다. 모바일에서는 썸네일과 제목을 먼저 보이게 하고, 채널·조회수·발행 시각은 한두 줄의 메타 정보로 접는다.
- 모바일 유튜브 필터는 검색어, 채널, 개수, 검색 버튼 순서의 단일 열로 둔다. 선택 작업은 별도 모드에서만 보여 목록 스캔을 방해하지 않는다.

## 뉴스 출처

- 운영 정책은 출처 리스트 상단에 둔다. 글로벌/코인 신규 출처 자동 노출과 숨김 포함 보기를 여기서 조정한다.
- 카테고리 전환은 출처 리스트 헤더의 탭으로 둔다.
- 출처 추가는 리스트 하단에 고정된 추가 영역에서 한다. `+` 버튼으로 입력 행을 늘리고, 저장 전 임시 행은 삭제할 수 있다.
- 기존 출처는 삭제보다 숨김/복구를 우선한다. 별칭은 출처별 모달에서 관리한다.

## 설정

- `Data Provider`에는 API Key와 활성 상태만 둔다. 수집 Provider에 모델 정보는 두지 않는다. 표시명은 `finnhub / youtube / ninjas / coingecko`처럼 내부 provider id와 맞춘다.
- LLM 모델 프리셋과 기본 모델은 `Provider > LLM Provider`에서 관리한다.
- 번역 설정은 로케일별 사용 여부, 자동 번역 여부, Provider 선택을 관리한다. 실제 모델은 선택된 Provider의 기본 모델을 따른다.
- 향후 뉴스 외 유튜브/기타 콘텐츠 번역이 늘어날 수 있으므로 번역 설정의 문구와 구조는 특정 도메인에 과하게 묶지 않는다.

## 오류와 로그

- 오류 로그 테이블은 요약 컬럼만 보여준다. 긴 오류 메시지는 `오류 확인` 모달에서 본다.
- 오류 모달은 메시지, 스택/원문, 실행 컨텍스트를 분리한다.
- 실행 이력 상세에서는 재시도나 삭제처럼 실제 운영 액션을 제공하고, 대시보드에서는 이동만 제공한다.
- 테이블 행의 경고/실패 강조는 행 전체를 강하게 물들이기보다 왼쪽 상태 라인과 낮은 채도의 배경으로 처리한다. 상태 자체는 pill로 한 번 더 표시한다.

## 다국어

- 새 UI 문자열은 `ko / en / ja` 키를 함께 추가한다.
- 화면 전환 없이 언어를 바꾸면 현재 화면의 동적 영역도 다시 렌더링되어야 한다.
- 영문 고유명사라도 메뉴나 설명성 라벨은 가능한 한 현재 언어에 맞춰 표시한다.

## 컴포넌트 기준

- 탭은 같은 화면의 범주 전환에만 쓴다. 상태 필터에는 컴팩트 탭을 사용할 수 있다.
- 테이블 액션은 우측 끝에 모으고, 동일 테이블 안에서는 액션 순서를 유지한다.
- 모달은 제목, 보조 메타, 본문, 하단 액션 순서로 구성한다. 닫기, 취소, Escape, backdrop 클릭을 지원한다.
- 버튼 텍스트는 짧게 쓴다. 익숙한 조작은 아이콘 버튼을 쓰되 `title`을 제공한다.
- 버튼 색상은 의미별로 고정한다. 저장·즉시 실행처럼 확정적인 primary action은 accent/`success`, 목록 이동·상세·검색·초기화는 `secondary`, 재시도·재번역·선택 갱신처럼 주의가 필요한 비파괴 액션은 낮은 채도의 `warning`, 삭제·초기화·로그아웃처럼 되돌리기 어려운 액션은 `danger`를 쓴다.
- 테이블 안의 버튼은 `compactBtn` 또는 테이블 액션 전용 크기를 써서 행 높이를 과하게 키우지 않는다.
- 모바일/좁은 화면에서는 필터와 그리드를 한 열로 내려 정보가 겹치지 않게 한다.
